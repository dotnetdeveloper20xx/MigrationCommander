Design Patterns

6 enterprise design patterns implemented with purpose

Factory Pattern

MigrationProviderFactory dynamically creates database-specific providers based on ProviderType enum.

Strategy Pattern

BaseMigrationProvider with database-specific implementations for SQL Server, PostgreSQL, MySQL, SQLite.

Observer Pattern

MigrationExecutorService fires events (ProgressChanged, Completed, Failed) for real-time UI updates.

Repository Pattern

DatabaseRepository abstracts data access with domain/entity separation and DPAPI encryption.

Builder Pattern

AddMigrationCommander(options) fluent configuration for services, database path, real-time updates.

Decorator Pattern

SignalRMigrationNotifier decorates NullMigrationNotifier to add real-time WebSocket notifications.

Quick Start

Get up and running in minutes - no Docker required

Terminal
# Clone the repository
git clone https://github.com/dotnetdeveloper20xx/MigrationCommander.git
cd MigrationCommander

# Build the solution
dotnet build

# Run tests (48 tests)
dotnet test

# Run the dashboard
dotnet run --project src/MigrationCommander.Dashboard

# Navigate to http://localhost:5035
Program.cs - Integration
// Add MigrationCommander services
builder.Services.AddMigrationCommander(options =>
{
    // In-memory database - no external dependencies
    options.InternalDatabasePath = "Data Source=MigrationCommander;Mode=Memory;Cache=Shared";
    options.EnableRealTimeUpdates = true;
});

// Add SignalR for real-time updates
builder.Services.AddMigrationCommanderSignalR();

// Configure middleware
app.UseMigrationCommander(seedTestData: true);
app.MapMigrationCommanderHub();
